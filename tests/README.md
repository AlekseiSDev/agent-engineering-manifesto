# Adoption Tests

This directory contains usage-based adoption tests for the manifesto.

The goal is not to prove correctness with unit assertions.
The goal is to see what an agent actually creates when asked to configure a fresh repository according to the manifesto.

Contents:

- `fixtures/`
  - fake repositories used as adoption targets
- `evals/adoption_checklist.md`
  - human-first rubric for evaluating the result
- `runs/`
  - prepared workspaces and prompts generated by `scripts/run-adoption-eval.sh`

Typical flow:

1. Prepare a run for a fixture.
2. Launch a subagent inside the prepared workspace.
3. Give it the generated prompt.
4. Evaluate the resulting repo with the checklist.
